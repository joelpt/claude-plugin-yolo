# D3 Earth Globe — Build Tasks

A progressive-enhancement build of a D3.js-based animated 3D rotating Earth.
Each task should be verified by opening `globe.html` in a browser before moving on.

- [ ] Create `globe.html`: embed D3 v7 from CDN; render a basic black sphere (orthographic projection, radius 250) centered on a 600×600 canvas with a dark (#0a0a1a) background and a subtle white sphere outline.
- [ ] Add land polygons: fetch Natural Earth 110m TopoJSON from CDN and draw filled countries in dark green (#1a3a1a) with slightly lighter borders (#2a5a2a), rotating continuously at 0.3°/frame on the Y axis.
- [ ] Texture the sphere: load NASA Blue Marble satellite imagery as a canvas texture tile; map it onto the globe using D3's `context.drawImage` clip path technique so the image conforms to the sphere outline.
- [ ] Add a cloud layer: load a NASA cloud PNG as a second canvas layer rendered 2px larger than the globe radius, with 0.6 opacity and rotating at 0.35°/frame (slightly faster than land) to simulate independent cloud drift.
- [ ] Add cloud shadows: render the cloud PNG a third time offset 8px down-right at 0.15 opacity in black, positioned between the surface and the cloud layer, creating a soft shadow effect.
- [ ] Add a starfield background: before drawing the globe each frame, scatter 800 white dots at random (x, y) in a stable seeded array; vary radius 0.4–1.2px and opacity 0.3–1.0 using a simple LCG seed so the star positions are deterministic across refreshes.
- [ ] Add the Moon: draw a small grey circle (radius 18px) orbiting the globe at 380px from center, completing one orbit every 10 seconds; add a subtle radial gradient (light grey to dark grey) to give it depth.
- [ ] Add a thin atmosphere glow: before drawing land, fill the sphere with a radial gradient from transparent at center to rgba(100,160,255,0.25) at the edge, giving the limb a blue halo effect.
- [ ] Add interactive rotation controls: on mouse drag, rotate the globe by converting the drag delta to longitude/latitude using `d3.drag()`; on scroll wheel, zoom the orthographic scale between 150 and 500; keep the animation running during interaction (merge drag rotation into the animation offset).
- [ ] Add an optional day/night terminator: compute the subsolar point for the current UTC time; draw a clip-path half-sphere darkened to rgba(0,0,30,0.55) representing the night side; update it once per second.
- [ ] Add a HUD overlay: bottom-left corner, show current UTC time (hh:mm:ss) and the lat/lon of the center of the visible face; update both at 1 Hz; style in monospace white on semi-transparent dark background.
- [ ] Smooth the rotation with momentum: on drag release, compute the velocity from the last two drag events and apply exponential decay (damping factor 0.95/frame) so the globe coasts to a stop rather than freezing instantly.
- [ ] Create `justfile` at `yolo-test/justfile` with recipes: `just open` (opens globe.html in the default browser), `just serve` (starts `python3 -m http.server 8080` in the background and opens `http://localhost:8080/globe.html`), `just validate` (runs a headless Chrome screenshot via `chromium --headless --screenshot=globe.png globe.html 2>/dev/null || python3 -m webbrowser globe.html` as a smoke check).
- [ ] Add a README.md in `yolo-test/`: document what was built, how to open it, and a one-paragraph description of each major visual layer (surface, clouds, stars, moon, atmosphere, terminator, HUD).
- [ ] Final polish pass: ensure the globe renders at 60 fps on a modern laptop (use `requestAnimationFrame` with `performance.now()` delta capping at 50ms); add a `<title>` tag; ensure the HTML is self-contained (no external files other than CDN scripts and NASA image URLs); validate the HTML with the W3C validator snippet `python3 -c "import urllib.request; print(urllib.request.urlopen('https://validator.w3.org/nu/?doc='+__import__('urllib.parse').quote('data:text/html,'+open('globe.html').read())).read().decode()[:200])"` and confirm no fatal errors.
